El problema oculto

Claude re-envía TODA la conversación en cada mensaje. Cada mensaje acumula el histórico completo.

MensajeTokens procesadosCoste real
1~500500 tokens
5~3,0003x más
15~15,000-25,00030x más
30~50,000+100x más

Las 7 reglas de oro

Regla 1: Nueva conversación cada 15-20 mensajes

El impacto más alto. Reinicia el contexto a cero. "fix the bug on line 42" en mensaje 30 puede costar 50,000+ tokens.

Regla 2: Modelo correcto por tarea

ModeloPrecio/MUso
Haiku 4.5$1 in / $5 outClasificar, extracción, fixes rápidos, boilerplate
Sonnet 4.6$3 in / $15 out80% del trabajo: código, multi-archivo, APIs
Opus 4.6$15 in / $75 outArquitectura compleja, debugging profundo, reviews

Regla 3: Contexto estructurado con XML

Claude responde mejor a contexto estructurado que a contexto largo.

<task>Fix race condition in token refresh</task>
<relevant_code>...solo código relevante...</relevant_code>
<error>...solo el error...</error>

Regla 4: Desactivar tools no usadas

Cada MCP connector carga su full definition en cada mensaje. 5 conectores sin usar = miles de tokens desperdiciados.

Regla 5: Enfoque por fases para proyectos grandes

Architecture (Opus) → Core (Sonnet, chat nuevo) → Modules (Sonnet, chat nuevo por módulo) → Review (Opus)

Regla 6: Prompt caching (API)

Cache read tokens cuestan 0.1x — ahorro de hasta 90% en contexto repetido.

Regla 7: Batch API para trabajo no urgente

50% descuento en todos los modelos. Para procesar archivos grandes o code reviews en batch.
