📝 Mejoras Reales

🔴 Urgentes

  • SSL: Verificar si funciona en https:// (Dani lo revisa en su navegador)
  • 5 páginas 404: Redirigir o crear contenido para las URLs que faltan:
    • /sobre-nosotros/ — Crear página "Quiénes somos"
    • /proyectos/ — Crear portfolio real
    • /que-es-bim/ — Crear página informativa BIM

🟠 Altas

  • Meta description: Diferentes para cada página
  • /escaner-laser/ y /inspiracion/: Reemplazar Lorem Ipsum con contenido real
  • Portfolio: Añadir proyectos reales (si los hay)

🟡 Medias

  • Schema markup (ProfessionalService)
  • Canonical URLs (www vs non-www)
  • sitemap.xml verificar que esté actualizado

📋 Migración a HTML — ¿Sí o No?

Argumentos PARA migrar:

  • WordPress con JS pesado = lento
  • Mantenimiento de updates constante
  • 5 páginas 404 = mal SEO
  • Lorem Ipsum visible = mala imagen

Argumentos EN CONTRA:

  • Si la web funciona bien y Dani está contento...
  • El contenido real es bueno
  • Hay redes sociales enlazadas

Recomendación: Arreglar primero los problemas concretos (404s, Lorem Ipsum, SSL si procede). Si después el mantenimiento de WordPress sigue siendo un coñazo, migrar a 11ty tiene sentido.
