Control UI — Official

URL: docs.openclaw.ai/web/control-ui Puerto default: http://127.0.0.1:18789/

Pages/Features:

| Feature | Description | |---------|-------------| | Chat | Message input, Markdown, code blocks | | Sessions | List, details, reset, delete, export | | Config | Edit config online, validation, hot reload | | Nodes | List connected devices, status, pairing | | Logs | Real-time streaming, filter, search, export | | Skills | Install from ClawHub, configure, toggle |

Config options:

{
  "web": {
    "enabled": true,
    "port": 18789,
    "bind": "loopback",
    "auth": {
      "mode": "password",
      "password": "your-secure-password"
    },
    "theme": "dark"
  }
}

Auth modes:

  • Token — via connect.params.auth.token
  • Password — gateway.auth.mode: "password"
  • Tailscale — identity headers when gateway.auth.allowTailscale: true
  • Trusted-proxy — identity headers from proxy

Remote access:

  • Tailscale: https://your-tailnet-name.ts.net
  • SSH tunnel: ssh -L 18789:localhost:18789 user@remote-server
  • Reverse proxy: Caddy/Nginx con reverse_proxy localhost:18789

1. openclaw-dashboard ⭐⭐⭐

GitHub: github.com/mudrii/openclaw-dashboard Autor: mudrii

#### 12 Paneles:

  • Top Metrics Bar — CPU, RAM, swap, disk + OpenClaw version
  • Header Bar — Bot name, online/offline, auto-refresh countdown
  • Alerts Banner — High costs, failed crons, gateway offline
  • System Health — Gateway status, PID, uptime, memory, sessions
  • Cost Cards — Today's cost, all-time, projected monthly, breakdown
  • Cron Jobs — All scheduled jobs with status/schedule/last-next
  • Active Sessions — Recent sessions con model, type badges
  • Token Usage & Cost — Per-model breakdown, 7d/30d/all-time
  • Sub-Agent Activity — Runs con cost, duration, status
  • Charts & Trends — Cost trend line, model breakdown
  • Bottom Row — Available models, skills list, git log
  • AI Chat — Natural language queries sobre dashboard
#### Características:
  • 6 built-in themes (3 dark + 3 light)
  • Glass Morphism UI
  • Responsive (desktop/tablet/mobile)
  • Local only — no external deps
  • Rate limiting en /api/chat
  • Go backend + pure HTML/CSS/JS frontend
#### Install:

Embed HTML en Control UI

Building the UI

La documentación menciona que el Gateway sirve static files desde dist/control-ui.

Para hacer embed de contenido custom en el Control UI:

  • Crear archivo HTML en el canvas root
  • Presentar via Canvas tool — canvas.present
  • O navegar — canvas.navigate

Hosted Embeds

Desde docs.openclaw.ai: 
El Control UI soporta hosted embeds vía:

  • /__openclaw__/canvas/.html
    • 

  • Custom URL scheme: openclaw-canvas:///
    •
